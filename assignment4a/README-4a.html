Image Crawler Reactive Streams App With Local Transforms (Assignment 4a)

The purpose of this assignment is to deepen your experience using Java's functional programming features and Java's reactive streams. In particular, in this assignment you'll develop an app that uses the RxJava implementation of the reactive streams specification to obtain, transform, and store images asynchronously in a pool of threads.

Like the earlier assignments, the code you'll write has a GUI and runs as an Android app. Please use this GUI along with the unit tests discussed below to help test and debug your solution.

Assignment Structure and Functionality

The ImageCrawler app is packaged as a project using the latest version of Android Studio. This app is written in both Kotlin and Java and demonstrates many Android capabilities. For the purposes of Assignment 4a, however, you only need to be aware of the following directories:

  • image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers, which contains the skeleton(s) you'll need to fill in, as described below.
  • image-crawler/src/test, which contains a set of unit tests that exercise the features you're implementing and are used to evaluate whether you've correctly satisfied this assignment's requirements.

To compile this code you need to use the provided Android Studio project. You can run this project by clicking the green "Run 'app'" button in the Android Studio IDE, which should automatically select an Android Emulator to run, assuming you have one created. The app's minimum API is 31, so you'll need to select an emulator that uses an API >= 31 (I recommend API 32). If you don't already have an Android Emulator you can create one by clicking on the "AVD Manager" button in the Android Studio IDE.

Program Description and "TODO" Tasks

You'll need to modify several files containing the skeleton Java code by completing the "TODO - You fill in here" tasks to provide a working solution. DO NOT CHANGE THE OVERALL STRUCTURE OF THE SKELETON - just fill in the "TODO" tasks (and do NOT delete the "TODO" markers)!!! In particular, you'll need to finish implementing the following "TODO" tasks for this assignment in the imagecrawler folder:

  • crawlers/RxObservableCrawler.java. You'll need to complete the "TODO" tasks in various methods to obtain, transform, and store images. This class uses Observable operators in RxJava to perform an "image crawl" asynchronously, starting from a root Uri. Depending on the parameters used to run the app, images from HTML pages reachable from the root Uri are either downloaded from a remote web server or read from the local file system, transformed locally, and the transformed results are then stored in the local file system on the Android device.

  • common/Student.kt. You'll need to complete the "TODO" to set the type field to include just Graduate or Undergraduate, depending on which version of the assignment that you're implementing.

The bulk of the code for the app is provided in the skeletons. In fact, a complete solution using Java 7 features is provided in the SequentialLoopsCrawler.java file. You will therefore convert this Java 7-based class to the RxObservableCrawler class that uses RxJava.

Your solution should use no loops or if/else statements in this assignment and instead use RxJava operators and functional programming features. Likewise, there should be only one call to blockingGet(). Your solution will be considered correct if it passes all the unit tests, is commented thoroughly, and addresses my review comments.
