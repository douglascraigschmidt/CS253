Image Crawler Parallel Stream Client App (Assignment 3b)

Part two of this assignment will deepen your experience using Java's functional programming features and their integration with Java's parallel streams by developing an app that uses Java parallel streams to obtain, transform, and store images. Please use this GUI along with the unit tests discussed below to help evaluate and debug your solution.

Assignment Structure and Functionality

The ImageCrawler app is packaged as a project using Android Studio. This app is written in both Kotlin and Java and demonstrates many Android capabilities. For the purposes of Assignment 3b, however, you only need to be aware of the following directories:

  • imagecrawler/crawlers, which contains the skeleton(s) you'll need to fill in, as described below.
  • image-crawler/src/test, which is a set of unit tests that exercise the features you're implementing and are used to evaluate whether you've correctly satisfied this assignment's requirements.

To compile this code you need to use the provided Android Studio project. You can run this project by clicking the green "Run 'app'" button in the Android Studio IDE, which should automatically select an Android Emulator to run, assuming you have one created. The app's minimum API is 31, so you'll need to select an emulator that uses an API >= 31 (I recommend API 34). You should already have Android Emulator installed, but if not you can create one by clicking on the "AVD Manager" button in the Android Studio IDE.

Program Description and "TODO" Tasks

You need to modify several files containing the skeleton Java code by completing the "TODO - You fill in here" tasks to provide a working solution. DO NOT CHANGE THE OVERALL STRUCTURE OF THE SKELETON - just fill in the "TODO" tasks (and do NOT delete the "TODO" markers or your code will not be reviewed)!!! In particular, you need to finish implementing the following "TODO" tasks for this assignment in the imagecrawler folder:

  • crawlers/ParallelStreamsCrawler.java. You need to complete the "TODO" tasks in various methods to obtain, transform, and store images via Java parallel streams that perform an "image crawl" starting from a root Uri. You should modify the SequentialStreamsCrawler class to use Java parallel streams, e.g., by replacing flatMap() (and in some places filter()) with the mapMulti() idiom. Images from HTML pages reachable from the root Uri are either downloaded from a remote web server or read from the local file system and the results are stored in the Android device's local file system.

  • common/Student.kt. You'll need to complete the "TODO" task to set the type field to include just Graduate or Undergraduate, depending on which version of the assignment that you're implementing.

Your solution should use no loops and (only a handful of if statements in the context of the mapMulti() idiom) in this assignment and instead use Java parallel stream aggregate operations and functional programming features. Your solution will be considered correct if it passes all the unit tests, is commented thoroughly, and addresses my review comments.
