Image Crawler Parallel Stream Client App (Assignment 1b)

The purpose of this assignment is to deepen your experience using Java's functional programming features and their integration with Java's parallel streams. In particular, in this assignment you'll develop an app that uses Java parallel streams to obtain, transform, and store images.

Like the first part of this assignment, the code you'll write here has a GUI, runs as an Android client app, and performs all image processing locally on the client device. Please use this GUI along with the unit tests discussed below to help evaluate and debug your solution.

The following resources may be helpful in completing this assignment:

  • The book Modern Java in Action is a great resource to learn how to program Java streams.

Assignment Structure and Functionality

The ImageCrawler app is packaged as a project using Android Studio. This app is written in both Kotlin and Java and demonstrates many Android capabilities. For the purposes of Assignment 1b, however, you only need to be aware of the following directories:

  • imagecrawler/crawlers, which contains the skeletons you'll need to fill in, as described below.
  • image-crawler/src/test, which is a set of unit tests that exercise the features you're implementing and are used to evaluate whether you've correctly satisfied this assignment's requirements.
  • app/src/androidTest, which is an Android Studio instrumentation test that runs your app automatically.

To compile this code you need to use the provided Android Studio project. You can run this project by clicking the green "Run 'app'" button in the Android Studio IDE, which should automatically select an Android Emulator to run, assuming you have one created. The app's minimum API is 31, so you'll need to select an emulator that uses an API >= 31 (I recommend API 32). If you don't already have an Android Emulator you can create one by clicking on the "AVD Manager" button in the Android Studio IDE.

Program Description and "TODO" Tasks

You'll need to modify several files containing the skeleton Java code by completing the "TODO - You fill in here" tasks to provide a working solution. DO NOT CHANGE THE OVERALL STRUCTURE OF THE SKELETON - just fill in the "TODO" tasks (and do NOT delete the "TODO" tags)!!! In particular, you'll need to finish implementing the following "TODO" tasks for this assignment in the imagecrawler/crawlers folder:

  • imagecrawler/crawlers/ParallelStreamsCrawler.java. You'll need to complete the "TODO" tasks in various methods to obtain, transform, and store images. This class uses Java parallel streams to perform an "image crawl" starting from a root Uri. It applies a couple of minuscule tweaks to SequentialStreamsCrawler.java class to use Java parallel streams. Depending on the parameters used to run the tests, images from HTML pages reachable from the root Uri are either downloaded from a remote web server or read from the local file system and the results are stored in the local file system on the Android device.

  • Student.kt. You'll need to complete the "TODO" to set the type field to include just Graduate or Undergraduate, depending on which version of the assignment that you're implementing.

Your solution should use no loops or if/else statements in this assignment and instead use Java parallel stream aggregate operations and functional programming features. Your solution will be considered correct if it passes all the unit and instrumentation tests, is commented thoroughly, and addresses my review comments.

Unit Tests

The unit tests in the image-crawler/src/test folder are provided to increase our collective confidence that your implementation is working as expected. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the unit tests to detect problems in your code.
